# Job documentation

14 jobs are fully implemented; PUP exists as a 12-module scaffold only.
Each job folder contains a `states.md` describing the modes and state
cycling options exposed by `<JOB>_STATES.lua`.

> **PUP** ships a 20-line skeleton set file (`_master/sets/pup_sets.lua`
> says "job not currently played"). The module structure works so anyone
> who wants to flesh it out can use it as a starter — but no real sets
> or specialty logic.
